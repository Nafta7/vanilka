Installation

It's possible to download a zip file from the project's repository but we'll be showing the optimal workflow to work with this theme, which is through the command line.

Clone the repository:

cd path/to/wp-site/wp-content/themes
git clone https://github.com/Nefla/vanilka.git theme-name
cd theme-name

Install the dependencies:

npm install

Build the assets:

npm build

Developing

After installing the dependencies and building the assets, you're ready to start developing your theme. All of the following commands are simple npm scripts that are available inside the package.json file.

Watching assets

The following command will watch for any changes in the assets directory and compile the Stylesheet and/or JavaScript file. 

npm run watch

Building assets

When the theme is ready for production, run the following command to generate the minified version of the assets: 

npm run build:production

For development, run the following command:

npm run build:development

Enabling live-reload

To enable live-reload you have to provide your local dev-server address to npm's package.json file. Open the package.json file and edit the proxy field in the config section with your local dev-server address.

Example:

"config": {
  "proxy": "localhost:8000"
}

Now you can run npm start to let browser-sync do the live reload:

 npm start

Theme structure

The theme structure has the usual Wordpress theme files with a couple aditions.

assets directory holds the Sass stylesheet files (inside styles directory) and the JavaScript files (inside scripts directory).

lib is where the Wordpress theme features are enabled, which are later on included or not from the functions.php file. This directory should be expanded as you wish, if a Wordpress feature that you want is not available in the lib, create a new file and then include it from the functions.php and call it from the necessary hook. Just try to follow this pattern.

node_modules is where the Node.JS dependencies are installed, which is ignored by git through the .gitignore file.

partials are neatly organized by post, page and sections. Partials are most known as template-parts and are here to help you keep organized.

templates directory is where the custom templates lives.

functions.php has the Wordpress hooks and is used to select the Wordpress features enabled in the lib directory.

package.json is where the Node.JS dependencies and scripts are declared.

Be aware, the style.css file is not listed here because you should generate when you first install the theme (following the aforementioned instructions). This file is very important because it contains meta data that is necessary to activate the theme from the admin panel.

To better illustrate the theme structure:

themes/vanilka/           # Theme root
├── assets/               # Theme assets
│   ├── styles/           # Stylesheet files
│   │   └── main.scss     # Sass entry point file
│   └── scripts/          # JavaScript files
│       └── app.js        # JavaScript entry point file
├── lib/                  # Wordpress theme selected features
├── node_modules/         # Node.js packages
├── partials/             # Partial templates
├── templates/            # Custom Wordpress templates
├── functions.php         # Wordpress hooks, theme includes
└──  package.json         # Node.js dependencies and scripts
